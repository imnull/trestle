## 一、产品概述

### 1.1 产品定位

**CodePlan Proxy** 是一款面向开发者的本地 AI 代理管理工具，让用户能够：
- 统一管理多个 AI 模型/服务
- 灵活配置 API 转发规则
- 为不同客户端提供兼容的 API 接口
- 通过可视化界面轻松管理一切

**一句话描述**: "一个界面管理所有 AI 编程助手"

### 1.2 目标用户

| 用户类型 | 需求场景 |
|---------|---------|
| 个人开发者 | 在多个 AI 编程工具间切换，想统一管理 API Key |
| 订阅用户 | 有多种 AI 服务订阅 (Claude/GPT/本地模型)，想按需路由 |
| 隐私敏感用户 | 想本地运行、不泄露 API Key |
| AI 爱好者 | 想尝试不同模型，对比效果 |

### 1.3 核心价值

1. **统一入口** — 一个端口服务所有 AI 工具
2. **灵活路由** — 按需选择模型，智能负载均衡
3. **成本透明** — 实时查看 token 消耗
4. **隐私安全** — 本地运行，数据不出机器
5. **开箱即用** — 下载即用，无需服务器

## 三、核心功能

### 3.1 代理服务 (Server)

#### 3.1.1 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/v1/chat/completions` | POST | OpenAI 兼容接口 |
| `/v1/messages` | POST | Anthropic 兼容接口 |
| `/v1/models` | GET | 返回可用模型列表 |
| `/api/config` | GET/PUT | 配置管理 |
| `/api/status` | GET | 服务状态 + 统计 |
| `/api/logs` | GET | 请求日志 (分页) |
| `/api/providers` | GET/POST/PUT/DELETE | 上游服务商管理 |
| `/api/models` | GET/POST/PUT/DELETE | 模型映射管理 |

#### 3.1.2 路由规则

```toml
# 用户配置的路由规则
[[routes]]
pattern = "gpt-4*"          # 模型名匹配 (支持通配符)
provider = "openai"         # 上游服务商
model = "gpt-4-turbo"       # 实际使用的模型
priority = 1                # 优先级 (数字越小越优先)

[[routes]]
pattern = "claude-*"
provider = "anthropic"
model = "claude-3-5-sonnet-20241022"
priority = 1

[[routes]]
pattern = "local-*"
provider = "ollama"
model = "qwen2.5-coder:7b"
priority = 2
```

**路由逻辑**:
1. 用户请求 `model: "gpt-4-code"`
2. 匹配规则 `gpt-4*` → OpenAI
3. 实际调用 `gpt-4-turbo`
4. 返回响应

#### 3.1.3 请求转换

```
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│ 客户端请求    │ ───► │ 代理转换      │ ───► │ 上游服务     │
│              │      │              │      │              │
│ OpenAI 格式  │      │ 格式适配      │      │ OpenAI API   │
│              │      │ Header 注入   │      │ Anthropic API│
│              │      │ 日志记录      │      │ 本地模型     │
└──────────────┘      └──────────────┘      └──────────────┘
```

## 四、配置文件设计

### 4.1 文件位置

```
~/.config/codeplan-proxy/
├── config.toml        # 主配置
├── providers.toml     # 服务商配置
├── routes.toml        # 路由规则
└── logs/
    └── requests.db    # SQLite 请求日志
```

### 4.2 配置格式

#### config.toml
```toml
[server]
host = "localhost"
port = 9527
log_level = "info"

[ui]
theme = "system"
language = "zh-CN"
auto_start = true
minimize_to_tray = true

[logging]
retention_days = 7
max_request_body_size = "1MB"
```

#### providers.toml
```toml
[[providers]]
name = "openai"
type = "openai"
base_url = "https://api.openai.com/v1"
api_key = "sk-xxx"
enabled = true

[[providers]]
name = "anthropic"
type = "anthropic"
base_url = "https://api.anthropic.com"
api_key = "sk-ant-xxx"
enabled = true

[[providers]]
name = "ollama"
type = "openai-compatible"
base_url = "http://localhost:11434/v1"
enabled = true
```

#### routes.toml
```toml
[[routes]]
pattern = "gpt-4*"
provider = "openai"
model = "gpt-4-turbo"
priority = 1

[[routes]]
pattern = "claude-*"
provider = "anthropic"
model = "claude-3-5-sonnet-20241022"
priority = 1

[[routes]]
pattern = "local-*"
provider = "ollama"
model = "${model}"  # 保持原名
priority = 2

[[routes]]
pattern = "*"
provider = "openai"
model = "gpt-4o"
priority = 99
```

## 五、技术选型

| HTTP 框架 | Axum | 高性能、异步 |

| UI 库 | Ant Design / Shadcn | 组件库 |
| 状态管理 | Zustand | 轻量状态 |
| 图表 | Recharts | 数据可视化 |
